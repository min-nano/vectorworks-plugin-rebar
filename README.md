# vectorworks-plugin-rebar

基礎コンクリートの配筋情報を保持し、平面ビュー・3D ビュー・断面ビューポートに適切な配筋表現を描画する VectorWorks **プラグインオブジェクト（PIO）** スクリプトです。

- **スラブモード**: 配筋する平面を 3D パス（閉じた外形）で指定し、主筋（`D10@200` 等）・配力筋（`D13@150` 等）・主筋方向の角度を保持します。ダブル配筋では上端筋・下端筋それぞれの仕様を保持します。
- **梁モード**: 梁天端の中心線を 3D パスで指定し、断面サイズ（`150×450` 等）・上下端主筋（`2-D16` 等）・せん断補強筋（`D10@200` 等）を保持します。せん断補強筋は仕様先頭の脚数（`1-`/`2-`/`3-`）で配置を切り替えられます（後述）。

いずれのモードも:

- **平面ビュー**では配筋を 2D の線で描きます。
- **断面ビューポート**では、紙面方向の鉄筋を直線、紙面直交方向の鉄筋を × で表示する **2D コンポーネント**（VectorWorks 2019 以降の機能）を出力します。断面ビューポートの詳細設定で「**2D コンポーネントを表示**」を有効にしてください。
- **3D ビュー**用に鉄筋の 3D ポリゴンも描きます。

## インストール

後述の `main.py` を PIO のスクリプトとして登録すれば、初回リセット時に依存ライブラリも含めて自動的にインストールされるため、手動インストールは任意です。オフライン環境などで手動インストールする場合は、このリポジトリをクローンまたはダウンロードし、リポジトリのルートで以下を実行します。

**macOS**
```bash
pip install --target "$HOME/Library/Application Support/Vectorworks/2025/Python Externals" .
```

**Windows（コマンドプロンプト）**
```bat
pip install --target "%APPDATA%\Nemetschek\Vectorworks\2025\Python Externals" .
```

> VectorWorks のバージョンが異なる場合は `2025` の部分を実際のバージョン番号に置き換えてください。

## VectorWorks へのプラグイン登録

ツール > プラグイン > **プラグインマネージャ** で新規プラグインを作成します。

1. **種別**: 「3D パス図形」（3D Path Object）を選択し、名前を `配筋` にする。
2. **パラメータ**: 以下のパラメータを **名前を完全一致** させて追加する（名前は半角英数、別名は OIP に表示したい日本語ラベル）。

   | 名前 | 種別 | 既定値 | 別名の例 |
   |---|---|---|---|
   | `Mode` | ポップアップ（`スラブ`・`梁`） | `スラブ` | モード |
   | `MainBar` | 文字列 | `D10@200` | 主筋（下端） |
   | `DistBar` | 文字列 | `D13@150` | 配力筋（下端） |
   | `MainBarAngle` | 数値 | `0` | 主筋角度 |
   | `DoubleLayer` | ブール | オフ | ダブル配筋 |
   | `TopMainBar` | 文字列 | `D10@200` | 上端主筋 |
   | `TopDistBar` | 文字列 | `D13@150` | 上端配力筋 |
   | `SlabThickness` | 数値 | `150` | スラブ厚 |
   | `SectionSize` | 文字列 | `150×450` | 断面サイズ（梁） |
   | `TopBars` | 文字列 | `2-D16` | 上端筋（梁） |
   | `BottomBars` | 文字列 | `2-D16` | 下端筋（梁） |
   | `Stirrup` | 文字列 | `D10@200` | せん断補強筋（梁）。先頭に脚数 `1-`/`2-`/`3-` を付けて配置を切り替え（省略時は 2） |
   | `Cover` | 数値 | `40` | かぶり |
   | `MarkScale` | 数値 | `4` | 断面記号倍率 |

3. **スクリプト**: `main.py` の内容をそのまま Python スクリプトとして貼り付ける。
4. ワークスペースの編集でツールをパレットに追加する。

登録したスクリプトはリセットのたびに GitHub の `main` ブランチの最新コミットを確認し、インストール済みのコミットと異なれば Python Externals フォルダへ自動的に更新インストールしてから本体を実行します（更新後は VectorWorks を再起動しなくても次のリセットから新しいコードが使われます）。インターネットに接続できない場合はインストール済みのバージョンをそのまま実行します。

## 使い方

### スラブモード

1. `Mode` を `スラブ` にしてツールを選び、配筋する平面の外形を 3D パスで描く（パス平面＝**スラブ天端**）。
2. 主筋・配力筋の仕様（`D10@200` 形式）と主筋方向の角度を OIP で設定する。配力筋は主筋に直交します。
3. シングル配筋はスラブ厚の中央に配筋されます。`DoubleLayer` をオンにすると、`MainBar`/`DistBar` が**下端筋**、`TopMainBar`/`TopDistBar` が**上端筋**になり、かぶり・鉄筋径を考慮した高さに配筋されます。

### 梁モード

1. `Mode` を `梁` にして、梁天端の中心線を 3D パスで描く。
2. `SectionSize`（幅×せい）・`TopBars`/`BottomBars`（`2-D16` 形式）・`Stirrup`（`D10@200` 形式）・`Cover` を設定する。
3. 断面（横断面）ではせん断補強筋がかぶりを除いた矩形、主筋が × で描かれます。縦断面（側面図）では主筋が水平線、せん断補強筋が等ピッチの縦線で描かれます。

#### せん断補強筋の配置（脚数）とフック

`Stirrup` の先頭に脚数を付けて、せん断補強筋の配置を切り替えられます（脚数を省いた `D10@200` は `2-` と同じ扱い）。

| 指定例 | 配置 | フック |
| --- | --- | --- |
| `1-D10@250` | 断面中央に縦筋 1 本のみ | 上下端に 180° フック |
| `2-D10@250` | 四角状のあばら筋 | 上端両隅に 135° フック |
| `3-D10@250` | 四角のあばら筋 + 中央の縦筋 | あばら筋 135°・中央縦筋 180° |

フックは配筋標準図（**材種 SD295A を仮定**）に従い、鉄筋径に応じた曲げ内法半径と余長で描きます。曲げ内法直径は D16 以下で 3d、D19 以上で 4d（＝内法半径 1.5d／2d）、余長は 180° フックが 4d、135° フックが 6d です。フックは断面（横断面）の 2D コンポーネントと 3D の両方に反映されます（円弧は折れ線で近似）。縦筋 1 本のみ（`1-`）は上面から見ると点になるため、平面ビューには線を描きません。

> **1 本の梁は 1 直線（1 オブジェクト）でモデル化してください。** 断面ビューポートで「梁を横断した切断＝横断面（×・矩形）」「梁に沿った切断＝側面図（水平主筋＋縦スターラップ）」を正しく切り替えるには、梁が単一方向である必要があります。折れ線（コの字・矩形など向きの違う区間を含むパス）で 1 本にすると、2D コンポーネントの仕組み上（VW は切断方向のコンポーネントを位置に関係なく表示する）、横断面と側面図が両方表示されてしまいます。曲がる梁は区間ごとに別々の梁オブジェクトとして作成してください。

### 断面ビューポートでの表示

断面ビューポートの詳細設定で「**2D コンポーネントを表示**」（ハイブリッドオブジェクトの 2D コンポーネント表示）を有効にすると、切断方向に応じて「前後の断面」「左右の断面」の 2D コンポーネントが表示されます。2D コンポーネントはオブジェクトのローカル軸 6 方向にしか持てないため、**X 軸・Y 軸に沿った配筋・梁で正確な表現**になります（斜め方向は近い軸の表現で近似されます）。

### 作図クラス

すべての図形は **PIO 本体の描画クラス**に割り当てられ、線種・色・太さはそのクラス属性に従います。PIO 本体（配筋オブジェクト）に割り当てるクラスを変えることで、配筋表現のクラス・見た目を管理できます。

## 開発

テストは VectorWorks の公式 `vs.py` スタブをモック対象として `pytest` で実行します（`.github/workflows/test.yml` 参照）。

```bash
pip install -e ".[test]"
curl -fsSL https://raw.githubusercontent.com/Vectorworks/developer-scripting/main/Python/pages/files/vs.py -o tests/vs.py
mypy
pytest tests/ -v
```
