# vectorworks-plugin-rebar

スラブ・壁の断面(紙面)に現れる **餅網状の配筋** を、直線を 1 本引くだけで
注釈として描く VectorWorks **プラグインオブジェクト（PIO）** スクリプトです。
**2D 線形図形**（Linear plug-in object）として登録し、ユーザーが引いた直線
（スラブ・壁の面）から **かぶり分オフセット** した位置に、配筋の断面表現を
2D の線と記号で並べます。

- **紙面平行方向の鉄筋** … かぶり分オフセットした **2D 直線** で描きます。
- **紙面直交方向の鉄筋** … 端部が見える鉄筋として、呼び径に応じた **断面
  表示記号**（●／× 等、配筋標準図 KSE 2008「鉄筋の表示記号」）を、オフ
  セット線上に **ピッチ間隔** で並べます。

線をいちいち描かなくても、直線を 1 本引くだけで配筋記号が並びます。
すべて **2D 注釈**（ビューポート注釈または設計レイヤの 2D 図形）として描き、
3D・断面 2D コンポーネントは使いません（VectorWorks の作図特性との相性から
2D 注釈方式へ全面刷新しました）。

## 表示記号（配筋標準図 KSE 2008）

紙面直交方向の鉄筋の端部は、呼び径ごとに下表の表示記号で描きます。記号の
大きさは `呼び径 × MarkScale` を外径とした模式表現です。記号定義は姉妹
プロジェクト「[鉄筋（vectorworks-plugin-rebar-single）](https://github.com/min-nano/vectorworks-plugin-rebar-single)」の
`rebar/symbol.py` を踏襲しています。

| 呼び径 | 記号 | 呼び径 | 記号 |
|---|---|---|---|
| D10 | ●（塗り丸・小） | D29 | ⊗（リング＋×） |
| D13 | ×（斜め十字） | D32 | ◎（二重リング） |
| D16 | ⊘（リング＋斜線） | D35 | ⊕（リング＋十字） |
| D19 | ●（塗り丸・大） | D38 | ●⊕（塗り丸＋十字） |
| D22 | ○（リング） | D41 | ⊗（リング＋×） |
| D25 | ⊙（リング＋中心点） | | |

## インストール

後述の `main.py` を PIO のスクリプトとして登録すれば、初回リセット時に依存
ライブラリも含めて自動的にインストールされるため、手動インストールは任意
です。オフライン環境などで手動インストールする場合は、このリポジトリを
クローンまたはダウンロードし、リポジトリのルートで以下を実行します。

**macOS**
```bash
pip install --target "$HOME/Library/Application Support/Vectorworks/2025/Python Externals" .
```

**Windows（コマンドプロンプト）**
```bat
pip install --target "%APPDATA%\Nemetschek\Vectorworks\2025\Python Externals" .
```

> VectorWorks のバージョンが異なる場合は `2025` の部分を実際のバージョン番号に置き換えてください。

## VectorWorks へのプラグイン登録

ツール > プラグイン > **プラグインマネージャ** で新規プラグインを作成します。

1. **種別**: 「**線形**」（Linear、2D 線形図形）を選択し、名前を `配筋` にする。
2. **パラメータ**: 以下のパラメータを **名前を完全一致** させて追加する（名前は半角英数、別名は OIP に表示したい日本語ラベル）。

   | 名前 | 種別 | 既定値 | 別名の例 |
   |---|---|---|---|
   | `ParallelBar` | 文字列 | `D10` | 紙面平行方向の鉄筋（線） |
   | `PerpBar` | 文字列 | `D13@200` | 紙面直交方向の鉄筋（記号） |
   | `Cover` | 数値 | `40` | かぶり |
   | `MarkScale` | 数値 | `4` | 断面記号倍率 |
   | `Flip` | ブール | オフ | オフセット反転 |

3. **スクリプト**: `main.py` の内容をそのまま Python スクリプトとして貼り付ける。
4. ワークスペースの編集でツールをパレットに追加する。

登録したスクリプトはリセットのたびに GitHub の `main` ブランチの最新コミット
を確認し、インストール済みのコミットと異なれば Python Externals フォルダへ
自動的に更新インストールしてから本体を実行します（更新後は VectorWorks を
再起動しなくても次のリセットから新しいコードが使われます）。インターネット
に接続できない場合はインストール済みのバージョンをそのまま実行します。

## 使い方

1. ツールを選び、スラブ・壁の面に沿って **直線を 1 本** 引く（2D 線形図形）。
   断面ビューポートの注釈、または設計レイヤの 2D として描けます。
2. `ParallelBar`（紙面平行方向の鉄筋、`D10` 形式の呼び径）を OIP で設定する。
   この鉄筋は、面線からかぶり分オフセットした 1 本の直線として描かれます。
3. `PerpBar`（紙面直交方向の鉄筋、`D13@200` 形式の呼び径＠ピッチ）を設定する。
   この鉄筋は、オフセット線上にピッチ間隔で並ぶ断面表示記号として描かれます。
4. `Cover`（かぶり、mm）を設定する。面線から `かぶり ＋ 平行筋径/2` だけ離れた
   位置に配筋が描かれます。
5. オフセットする側（面のどちら側に配筋を描くか）は面線の進行方向の左側が
   既定です。逆側にしたい場合は `Flip`（オフセット反転）をオンにします。

### オフセットの向き・記号の見え方について

面線からのオフセットの左右、記号の紙面上の大きさ・見え方は **VectorWorks
上で最終確認** してください（描画フェーズは VW 上で検証する方針です）。左右
が逆であれば `Flip` で反転できます。

紙面平行方向の鉄筋（線）と紙面直交方向の鉄筋（記号）は、簡潔さのため同一の
オフセット線上に模式化して描きます（餅網の 2 方向の層の微小な深さ差は現状
考慮しません）。上下端の 2 段配筋（ダブルメッシュ）は、面線を 2 本引いて
それぞれに配置する運用です。

### 作図クラス

すべての図形は **PIO 本体の描画クラス**に割り当てられ、線種・色・太さ・塗り
はそのクラス属性に従います。PIO 本体（配筋オブジェクト）に割り当てるクラス
を変えることで、配筋表現のクラス・見た目を管理できます。

## 開発

テストは VectorWorks の公式 `vs.py` スタブをモック対象として `pytest` で実行
します（`.github/workflows/test.yml` 参照）。

```bash
pip install -e ".[test]"
curl -fsSL https://raw.githubusercontent.com/Vectorworks/developer-scripting/main/Python/pages/files/vs.py -o tests/vs.py
mypy
pytest tests/ -v
```
